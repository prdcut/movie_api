FlixMe

The main purpose of this creation is to build the server-side component of a movies web application. The web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

Business Logic URL HTTP Method Request Response
Return a list of all movies. /movies GET None A JSON object holding data about all the movies. It contains title, genre, director, description and an imagePath.
Return data about a sinlge movie by title. /movies/:title GET None A JSON object holding data about the required movie. It contains title, genre, director, description and an imagePath.
Return data about a genre by name. /movies/genre/:name GET None A JSON object holding data about the required genre. It contains name and description.
Return data about a director by name. /movies/director/:name GET None A JSON object holding data about the required director. It contains name, short bio, birth year and death year.
Allow new users to register. /users POST A JSON object holding data about the user to be. A JSON object holding data about the user. It contains username, password, email and birthday.
Allow users to update their user info. /users/:Username PUT A JSON object holding data about the user to update. A JSON object holding the updated data about the user. It contains username, password, email and birthday.
Allow users to add a movie to their list of favorites. /users/:id/favorites/:Movie_ID PUT A JSON object holding data about the movie to add to the list of favorites. It contains the _id of the movie. A JSON object containing data about the updated user's "Favorites" list. It contains the current IDs of the movies from the user's list.
Allow users to remove a movie to their list of favorites. /users/:id/favorites/:Movie_ID DELETE None A JSON object containing data about the updated user's "Favorites" list. It contains the current IDs of the movies from the user's list.
Allow existing user to deregister. /users/:id DELETE None A message indicating the username was successfully removed from the database.
